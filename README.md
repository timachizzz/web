## 🎯 Суть проекта

Приложение отображает **SVG-карту парка**, на которой размечены деревья, образующие узнаваемый паттерн (надпись, символ). Пользователь регистрируется и выбирает свободное дерево, которое обязуется посадить лично. После бронирования дерево визуально меняет цвет и отображает имя владельца.

---

## 🗺 Логика карты и типы деревьев

### Структура SVG

SVG-файл парка разделён на две логические зоны:

| Зона | Тип дерева | Описание |
| :--- | :--- | :--- |
| **Буквы / Надпись** | Дуб | Деревья формируют текст (например, «ПОБЕДА») |
| **Сектора / Фон** | Плодоносные | Яблони, груши, сливы и т.п. — заполняют остальное пространство |

### Идентификация деревьев

Каждое дерево в SVG имеет уникальный `id`, по которому с ним можно взаимодействовать через JavaScript. Пример:

```html
<circle id="tree-42" data-type="oak" cx="150" cy="200" r="8" fill="green" />
```
Атрибуты:

    id — уникальный идентификатор

    data-type — тип (oak или fruit(С уточнением))

    fill — цвет (меняется динамически при бронировании)

## 🗄 Структура базы данных
    Таблица : FamilyUser
    Поле > Тип > Назначение
    id > INT > Первичный ключ
    surname > VARCHAR > Фамилия семьи
    family_size > INT >Количество членов семьи
    trees_alr_planted > VARCHAR > JSON-список id уже посаженных деревьев
    num_trees_alr_planted > INT > Счётчик посаженных деревьев
    Таблица : Tree
    Поле > Тип > Назначение
    id > INT > Первичный ключ
    name > VARCHAR > Название дерева 
    tree_type > VARCHAR > Тип: oak или fruit
    coordinates > VARCHAR > Координаты на SVG (x, y)
    sector_id > INT > FK на TreeSector
    taken_at > DATETIME > Дата бронирования
    planted_by > VARCHAR > Кем забронировано (фамилия)
    is_able_to_plant > BOOL > Доступно ли для бронирования
    Таблица : TreeSector
    Поле > Тип > Назначение
    id > INT > Первичный ключ
    type > VARCHAR > Тип сектора (oak / fruit_apple / fruit_pear / fruit_plum...)
    is_full > BOOL > Полностью ли занят сектор
## 🔌 API Эндпоинты
    Метод > URL > Что делает
    POST > /api/register > Регистрация семьи
    POST > /api/login > Вход
    GET > /api/logout > Выход
    GET > /api/trees > Получить JSON со всеми деревьями и их статусами
    POST > /api/trees/<id>/book > Забронировать дерево
    GET > /api/user/me > Получить данные текущего пользователя и его деревья
## 👤 Роли
Роль : Возможности
Гость : Просмотр SVG-карты, наведение на деревья
Семья (авторизованный) : Бронирование свободного дерева, просмотр своих деревьев

## 🖥 Сценарий работы

>    Семья заходит на сайт

>    Видит SVG-карту парка с цветными кругами-деревьями

>    Регистрируется (фамилия + размер семьи + пароль)

>    Наводит курсор на зелёный круг — видит подсказку: «Дуб черешчатый. Свободен»

>    Кликает → подтверждает бронирование

>    Круг становится серым с подписью фамилии семьи при наведении

>    В базе обновляются счётчики семьи и статус сектора
